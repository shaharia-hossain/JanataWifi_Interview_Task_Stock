# Stock Management System (Interview Task)

This repository contains the solution for a technical interview task focused on building a backend stock management system using Python. It highlights my ability to write clean, maintainable, and production-ready code under time constraints.

## 📝 Overview
The project involves designing a robust API to manage inventory items, track stock levels, and handle transactions. It demonstrates proficiency in API design, database interactions, and backend logic structuring.

## 🛠 Features
- **CRUD Operations:** Full Create, Read, Update, and Delete functionality for stock items.
- **Transaction Handling:** Secure endpoints for recording stock intake and dispatch.
- **Data Validation:** Ensuring data integrity before database persistence.
- **Scalable Architecture:** Designed with separation of concerns (Controllers, Services, Models).

## 💻 Tech Stack
- **Language:** Python
- **Framework:** Django / DRF (or equivalent framework used)
- **Database:** Relational SQL

## 🎯 Objective
To showcase practical software engineering skills, architectural planning, and adherence to best practices in REST API development.
